- 클래스와 멤버 변수의 접근 권한을 최소화하라고 했었다.
- 동일하게 지역 변수의 유효 범위를 최소한으로 줄이면 오류는 줄고 가독성과 유지 보수성은 늘어난다.

---

1. **지역 변수는 가장 처음 쓰일 때 선언하라.**

- 미리 선언을 하게 되면 코드가 어수선해지며 가독성이 떨어진다.
- 변수를 실제로 사용하는 시점에 타입이나 초기값이 생각나지 않을 수 있다.
- 지역 변수를 생각 없이 선언하면 너무 앞서 쓰여지거나, 쓰인 후에도 살아 있을 떄가 있다.
- 이 경우 실수로 변수를 중복 선언 하게 되면 디버깅이 매우 힘들어진다.

2. **거의 모든 지역 변수는 선언과 동시에 초기화 해야 한다.**
- 초기화에 필요한 정보가 다 모이기 전까지는 아예 선언을 해서는 안된다.
- 다만 try-catch문은 예외인데, 변수가 만약 초기화 단계에서 에러가 날 가능성이 있다면 오류의 전파를 막기 위해 try 구문 안에서 변수를 초기화 해야한다.
- try 블럭 바깥에서도 변수를 사용해야 한다면, 먼저 선언 후 try 구문 안에서 초기화 해야한다.

- for문이나 for-each문은, 반복 변수의 범위를 반복문 블럭 내부로 제한한다.
- 따라서 반복 변수 값을 반복문 바깥에서도 쓸게 아니라면 for문이 while문보다 낫다.
- 만약 while문을 사용하게 되다면, 아래와 같은 버그가 있을 수 있다.

```java
Iterator<Element> i = c.iterator();
while (i.hasNext()) {
	doSomething( i.nextO);
}
Iterator<Element> i2 = c2.iterator();
while (i.hasNext()) { // 버그!
	doSomethingElse(i2.next());
}
```
- 위 코드에서는 while문을 붙여 넣은 후 잘못 수정하여 아래 while문에서도 i2대신 i를 쓰고 있다.
- i는 초기화가 안되어 있기 때문에, c2를 제대로 순회하지 못한다.
- for문은 변수의 범위가 한정되어 있어 i를 여러번 사용해도 서로 영향이 없으며, 세련되고, 가독성도 좋다.

```java
for (int i = 0, n = expensiveComputation(); i < n; i++) {
... // i로 무언가를 한다.
}
```
- 이는 for문의 또 다른 사용법인데, 한계값을 결정하는 n을 맨 처음에 초기화해서 for문이 돌 때마다 값을 계산하지 않는다.
- 만약 같은 값을 반환하는 메서드를 매번 사용해야 한다면 이처럼 사용하자.


3. 메서드를 작게 유지하고 한가지 기능에 집중하게 만들자.
	- 한 메서드에서 여러가지 기능을 수행한다면 한 기능에서만 사용하는 변수에 다른 기능에서 접근할 수 있을 것이다.
	- 메서드를 가능한한 기능별로 쪼개자.
